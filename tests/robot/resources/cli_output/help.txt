*
Generate an interactive HTML dashboard from Robot Framework output.xml results.

options:
  -v, --version         Display application version information.
  -h, --help            Provide additional information.

output files:
  -o, --outputpath [[]PATH ...[]]
                        Path to one or more output.xml files.
                          • Repeat '-o' for multiple files
                          • Append ':tag1:tag2' to add tags
                        Examples:
                          • '-o path/to/output1.xml'
                          • '-o output2.xml:dev:nightly -o output3.xml:prod'
  -f, --outputfolderpath [[]PATH ...[]]
                        Directory scanned recursively for *output*.xml files.
                          • All matching files in subfolders are included
                          • Append ':tag1:tag2' to add tags
                        Examples:
                          • '-f results/'
                          • '-f results/ -f path/to/more_results/:prod:regression'
  --projectversion VERSION
                        Project version associated with the processed runs.
                          • Cannot be mixed with version_ output tags
                        Examples:
                          • '--projectversion=1.1'
  -z, --timezone OFFSET
                        UTC offset of the output.xml timestamps (default: auto-detected).
                          • Format: +HH:MM or -HH:MM
                          • Use '--timezone=' form for negative offsets
                        Examples:
                          • '-z +02:00'  '--timezone=-05:00'  '-z +00:00'

database:
  -d, --databasepath PATH
                        Path to the database file (default: robot_results.db).
                        Examples:
                          • '-d path/to/myresults.db'
  -r, --removeruns [[]QUERY ...[]]
                        Remove runs by index, run_start, alias, tag, or limit.
                          • Separate multiple values with commas; ranges use ':', lists use ';'
                        Examples:
                          • '-r index=0,index=1:4;9,index=10'
                          • '-r alias=some_alias,tag=prod'
                          • '-r limit=10' -> keep only the 10 most recent runs
  -c, --databaseclass PATH
                        Path to a custom database class (.py) to override the built-in SQLite engine.
                          • See docs for implementation details
                        Examples:
                          • '-c customdb.py'
  --novacuum [[]BOOL[]]     `boolean` [[]default: False[]] Disable automatic database vacuuming.

dashboard:
  -g, --generatedashboard [[]BOOL[]]
                        `boolean` [[]default: True[]] Generate the HTML dashboard file.
  -l, --listruns [[]BOOL[]]
                        `boolean` [[]default: True[]] List runs in the console output.
  -n, --namedashboard NAME
                        Custom HTML dashboard file name (default: robot_dashboard_yyyymmdd-hhssmm.html).
                        Examples:
                          • '-n dashboard.html'
  -t, --dashboardtitle TITLE
                        Custom HTML title (default: Robot Framework Dashboard - yyyy-mm-dd hh:mm:ss).
                        Examples:
                          • '-t My_Test_Dashboard'
  -q, --quantity INT    Number of runs shown on initial dashboard load (default: 20).
                          • Higher values slow initial load
                        Examples:
                          • '-q 25'
  --offlinedependencies [[]BOOL[]]
                        `boolean` [[]default: False[]] Embed JS/CSS locally instead of loading from CDN.

log linking:
  -u, --uselogs [[]BOOL[]]  `boolean` [[]default: False[]] Enable clickable graphs that open the corresponding log.html.
  --logurl URL          URL to the log file for the processed run(s), stored in the database.
                          • Overrides the path derived from the output.xml location
                          • Use '{run_alias}' placeholder for multi-file processing
                        Examples:
                          • '--logurl https://ci.example.com/build42/log.html'
                          • '--logurl https://ci.example.com/build42/log_{run_alias}.html'

dashboard configuration:
  -j, --jsonconfig PATH
                        Path to a JSON configuration file applied on first dashboard load.
                        Examples:
                          • '-j settings.json'
  --forcejsonconfig [[]BOOL[]]
                        `boolean` [[]default: False[]] Force --jsonconfig to override existing localstorage settings.
  -m, --messageconfig PATH
                        Path to a file containing test message templates with ${placeholder} syntax.
                        Examples:
                          • 'The test failed on: ${date}' -> example line
                          • '-m messages.txt'

server:
  -s, --server [[]HOST:PORT:USERNAME:PASSWORD[]]
                        Start the dashboard webserver.
                          • Format: 'default' or 'host:port', optionally ':username:password'
                        Examples:
                          • '--server'  '--server default'
                          • '--server 0.0.0.0:8080'
                          • '--server 0.0.0.0:8080:admin:secret'
  --noautoupdate [[]BOOL[]]
                        `boolean` [[]default: False[]] Disable automatic dashboard regeneration on upload/delete.
                          • When enabled, use the 'Refresh' buttons to refresh the dashboard
  --ssl-certfile PATH   Path to SSL certificate file to enable HTTPS (requires --ssl-keyfile).
                        Examples:
                          • '--ssl-certfile cert.pem --ssl-keyfile key.pem'
  --ssl-keyfile PATH    Path to SSL private key file to enable HTTPS (requires --ssl-certfile).
                        Examples:
                          • '--ssl-certfile cert.pem --ssl-keyfile key.pem'

Boolean flags: omit the value to toggle the default (e.g. '--uselogs'); or pass 'true'/'false' explicitly.
For full documentation, visit: https://marketsquare.github.io/robotframework-dashboard/
